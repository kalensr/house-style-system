# House Style System

The House Style System has three layers:

1. Written guidance in `HOUSE_STYLE.md`.
2. Automated checks through Vale.
3. Human review for truth, evidence, judgment, and tone.

The system works best when those layers stay separate.

## Operating Model

Draft naturally. Edit for substance. Apply the house style. Run the style gate before handoff. Fix warnings that weaken clarity or trust. Record intentional exceptions when needed.

The style gate is a checkpoint, not a writing process.

## What Automation Owns

Automation should catch objective or repeatable risks:

- em dashes,
- filler phrases,
- weak modifiers,
- jargon,
- stock conclusions,
- overloaded sentences,
- acronym candidates,
- repeated contrast scaffolding.

## What Human Review Owns

Human review still owns:

- factual accuracy,
- evidence quality,
- source fidelity,
- missing context,
- recommendation strength,
- reader fit,
- final accountability.

## A Useful Loop

When a pattern keeps showing up:

1. Name the pattern.
2. Add guidance.
3. Add a fixture.
4. Add or adjust a rule only when the pattern is checkable.
5. Run the test.
6. Keep the warning soft unless the issue is always wrong.

This keeps the system useful without turning it into a brittle compliance tool.
